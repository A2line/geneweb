GeneWeb - Séparer les données dans la base


Si vous avez reçu des fichiers GEDCOM ou des données d'autres chercheurs, ou si vous voulez que votre base de données soit séparée en plusieurs fichiers, mais que vous voulez n'avoir qu'une seule base de données, GeneWeb peut faire cela pour vous.

Fusionner des fichiers dans une base de données

  • Si vous avez trois fichiers sources "toto.gw" "dupont.gw" et "titi.gw" que vous voulez qu'ils restent séparés. Lancez une fenêtre de commandes intéractives et tapez: 
         gwc toto.gw dupont.gw titi.gw -o basefusionnee
    Il se peut que gwd affiche des erreurs parce que des personnes sont définies deux fois dans "toto.gw" "dupont.gw" et "titi.gw". Pour corriger ça, gwc a une option "-sh" pour décaler les numéros des personnes et éviter les conflits. Essayez: 
         gwc toto.gw -sh 10 dupont.gw -sh 10 titi.gw -o basefusionnee
    Si la valeur 10 ne résout pas le problème, essayez 20, 30, 100...
    Voir aussi Fusion de bases de données.

  • Maintenant vous pouvez accéder et mettre à jour la base de données "basefusionnee". GeneWeb se rappellera dans quels fichiers les données proviennent.

  • Dans les requêtes de la base de données, la séquence ";opt=from" peut être ajouté pour voir le nom du fichier d'où viennent les données. 
         http://machine:2317/basefusionnee?i=357;opt=from
    Note: Cela ne fonctionne que sur les personnes ayant des familles.

Pour recréer les fichiers sources à partir de la base fusionnée

  • Si le répertoire "outdir" n'existe pas, créez-le (avec la commande "mkdir")

  • Pour recréer les fichiers sources, tapez dans une fenêtre de commandes intéractives: 
         gwu basefusionnee -odir outdir
  • Dans le répertoire "outdir", les trois fichiers sources "toto.gw" "dupont.gw" et "titi.gw" sont recréés. Si vous avez changé votre base de données, ces trois fichiers sont aussi mis à jours (c'est-à-dire qu'ils ne sont pas les mêmes que ceux du départ).

    Si la commande gwu affiche du texte source

  • Alors cela signifie que vous avez ajouté des personnes que GeneWeb ne sait pas dans quel fichier source elles doivent être (par exemple si ce sont de nouvelles personnes reliées à aucune autre personne des fichiers initiaux). GeneWeb redirigera ces personnes dans la fenêtre de commandes intéractives.

  • Pour sauver aussi ces personnes dans le fichier source "nouveau.gw", tapez: 
         gwu basefusionnee -odir outdir > nouveau.gw
  • Le contenu de "nouveau.gw" peut alors être transféré ou réparti dans les autres fichiers sources avec un éditeur de textes.
