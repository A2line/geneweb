Paramètres "$a"

Configurez les paramètres de la base $a:

body_prop Permet de configurer les couleurs de fond, ou d'y mettre un motif et de changer la couleur du texte et des liens. Sa syntaxe est la syntaxe HTML acceptée dans le tag "<body>" avant le ">". Voyez votre documentation HTML. Exemple: 

     background=fond.jpg text=#CC0000 vlink=yellow

Quelques variables intéressantes...

  • background: image de fond
  • bgcolor: couleur de fond
  • text: couleur du texte
  • link: couleur des liens
  • vlink: couleur des liens déjà visités

Ceci s'applique non seulement à la page d'accueil mais à toutes les pages affichées de cette base.

default_lang Langue par défaut.

Si c'est "-", c'est la langue par défaut de la commande "gwd" qui sera prise.

 

friend_passwd Mot de passe "ami".

Les "amis" peuvent voir toutes les dates, notes et photos des personnes nées depuis moins d'un siècle. Ceux qui n'ont pas entré le mot de passe ci-dessous ne verront pas ces informations.

Si cette zone est vide, tout le monde est "ami".

 

wizard_passwd Mot de passe "sorcier".

Les "sorciers" peuvent en outre faire des modifications dans la base de données.

Si cette zone est vide, tout le monde est "sorcier".

 

can_send_image Possibilité d'envoyer des images.

Si oui, les "sorciers" ont l'autorisation d'envoyer des images par le réseau.

Peut être utile en local pour pouvoir mettre une image à des personnes, car la procédure installe automatiquement la photo au bon endroit avec le nom approprié.

  oui non

wizard_just_friend Les "sorciers" deviennent de simples "amis".

Si oui, les "sorciers" perdent leur pouvoir de mise-à-jour.

Cela peut-être utile, temporairement, quand on fait un nettoyage de la base, si elle est accessible sur Internet, pour éviter que des "sorciers" ne fassent des mises-à-jour qui seraient perdues pendant l'opération (penser alors à supprimer cette option après nettoyage).

  oui non

renamed Base renommée.

Si vous avez changé le nom de votre base, mais que c'est une base qui existe depuis longtemps et que vous pensez que beaucoup de gens ont mis des liens dessus, conservez ce fichier de paramètres pour cette base et indiquez ci-dessous le nouveau nom de la base. Les gens qui consulteront la base avec l'ancien nom recevront une page qui leur indiquera le nouveau nom à utiliser avec leur même requête cliquable.

Si cette zone est vide, la base n'est pas renommée.
